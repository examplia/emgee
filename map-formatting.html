Map Formatting

General

  • Maps follow a strict format in order for them to be properly loaded in the game.
  • You will NOT be able to load your map if this format is not PERFECTLY followed.
  • Some folders are not absolutely necessary for your map to load, however it is highly recommended to include every folder in your map, whether it is being used or not.
  • Folders and models can be used interchangeably, so use whichever you prefer when building your map!

Tips

  • Keep a copy of the test map for your map’s gamemode at all times in your workspace! Once you’ve finished the geometry (or Ignore, Doors, etc.), simply replace the test-map parts with your own.
  • Play around on the test maps in-game and see for yourself what you can and can’t shoot through, what opens and closes, what you can and can’t walk on, and whatever else interesting you might find!
  • Follow along with the test maps and prop pack when reading this guide!
  • USE THE TEST MAPS!!!

Geometry

  • The Geometry folder contains most of the physical environment that players see and interact with.
  • Without it, players wouldn't have a map to even play on!
  • All parts in Geometry block hitscan (bullets and melee).
  • Disabling CanCollide allows projectiles and players to pass through the part.
  • Disabling CanTouch allows stickies to bounce off the part.
  • The only parts not included in Geometry are player interactables (Doors, health/ammo kits, KillBricks) and parts that do not block hitscan.

Ignore

  • The Ignore folder contains important gameplay features: CameraSpots, KillBricks, NoBuilds, and Water.
  • Parts in Ignore are completely ignored by hitscan and projectiles, excluding Annihilator's projectiles, which behave just as they do in Geometry.
  • CameraSpots contain all of the spectator camera positions for each team. There are three folders inside: General, GRN, and RED.
    • General allows both teams to use the camera.
    • GRN and RED allow only their respective teams.
    • Cameras are created by placing a part inside those subfolders; add a NumberValue called “FOV” to modify field of view.
  • KillBricks kill the player instantly upon contact.
  • NoBuilds prevent building within their bounds.
  • Water turns all parts into Roblox terrain water.

Doors

  • The Doors folder contains all of the doors that open and close depending on player interaction and which points have been captured.
  • Door models have two core attributes:
    • Move: how many studs in each direction the door moves when opened.
    • MoveTime: how long (in seconds) it takes to move.
  • Door parts behave the same as parts in Geometry (collisions, projectiles, etc.).
  • Each door model must contain:
    • PrimaryPart (under Pivot)—moves when the door opens/closes; must be Anchored. See photos on left.
    • Trigger—detects player input to open. Configuration named “Properties” with:
      • ActivationType (1–4):
        • 1 = touch (disables CanCollide, acts like Ignore).
        • 2 = melee (enables CanCollide, acts like Geometry).
        • 3 = melee, bullets, projectiles (enables CanCollide).
      • Duration: how long the door stays open (seconds).
      • PressMax: uses per round (–1 = unlimited).
      • Team: “GRN”, “RED”, or blank.
      • WaitForFinish: forces wait until door closes before re-trigger.
    • ForceCloseDuring—stages the door cannot be opened (Setup, Stage1, Stage2, Stage3).
    • ForceOpenDuring—stages the door stays permanently open; cannot overlap with ForceCloseDuring.

Clips

  • The Clips folder contains all invisible walls that block players.
  • Parts in Clips are ignored by weapons but block players.
  • CanCollide must be enabled; other part properties obsolete.

Items

  • The Items folder contains resupply cabinets and health/ammo kits.
  • Models match in-game sizes—use them!
  • Parts named “Resupply” spawn cabinets; named “SizeType” spawn kits (e.g. “MediumAmmo”, “SmallHP”).

ControlPoints

  • Contains control points for GameModes: KOTH, AD, 3CP, Arena.
  • KOTH/Arena: folder named “ControlPoint” with the hill.
  • AD: ControlPoint1 (first), ControlPoint2 (unlocked after CP1).
  • 3CP: CP1=Red start, CP2=neutral, CP3=Green start.
  • Each folder contains PlayerCaptureTime, PointLocation, CaptureZone.

Briefcases

  • Folder for CTF objectives only: “Green” & “Red.”
  • Objectives can be any part named accordingly.
  • GreenTrigger / RedTrigger fire on touch by opposing carrier.

TeamSpecific

  • Subfolders: GRN & RED containing Spawns, Checkpoints, SpawnZones, Barriers.
  • Spawns: respawn points per stage.
  • Checkpoints: optional; respawn at last touched.
  • SpawnZones: class/loadout zones; opposing zone = kill.
  • Barriers: block only opposite team.

Settings

  • Folder contains Lighting, Terrain, and General settings.
  • Use the TC2 Map Plugin to automate Lighting & Terrain.
  • General toggles (AllowUnanchoredParts, Climbing) off unless needed.

GameMode

  • GameMode StringValue sets the gamemode: "KOTH", "AD", "3CP", "CTF", "Arena", "None".
  • KOTH: adds ActivateIn IntValue.
  • AD: adds GameTimer, FirstCap, SetupTimer.
  • 3CP: no extra values.
  • CTF: adds CritsOnCap, MaxTime, PlayTo, TimeOnCap.
  • Arena: adds ActivateIn, PrepTime.
  • None: adds PrepTime (infinite timer; end via VIP command).
