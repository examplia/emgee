Map Formatting

General

  • Maps follow a strict format in order for them to be properly loaded in the game.
  • You will NOT be able to load your map if this format is not PERFECTLY followed.
  • Some folders are not absolutely necessary for your map to load, however it is highly recommended to include every folder in your map, whether it is being used or not.
  • Folders and models can be used interchangeably, so use whichever you prefer when building your map!

Tips

  • Keep a copy of the test map for your map’s gamemode at all times in your workspace. Once you’ve finished the geometry (or Ignore, Doors, etc.), simply replace the test-map parts with your own.
  • Play around on the test maps in-game to discover shoot-through walls, open/close doors, walkable surfaces, and anything else interesting.
  • Follow along with the test maps and prop pack when reading this guide.
  • USE THE TEST MAPS!!!

Geometry

  • The Geometry folder contains most of the physical environment that players see and interact with.
  • Without it, players wouldn't have a map to even play on!
  • All parts in Geometry block hitscan (bullets and melee).
  • Disabling CanCollide allows projectiles and players to pass through the part.
  • Disabling CanTouch allows stickies to bounce off parts.
  • The only parts not included are player interactables (Doors, kits, KillBricks) and non-blocking parts.

Ignore

  • The Ignore folder contains CameraSpots, KillBricks, NoBuilds, and Water.
  • Parts in Ignore are ignored by hitscan and projectiles (except Annihilator’s).
  • CameraSpots holds spectator positions—subfolders: General, GRN, RED.
  • KillBricks kill players on contact.
  • NoBuilds prevent building within bounds.
  • Water turns parts into terrain water.

Doors

  • The Doors folder contains all of the doors that open and close depending on player interaction and which points have been captured.
  • Door models have two core attributes:
    • Move: how many studs in each direction the door will move when opened.
    • MoveTime: how long (in seconds) it takes to move the distance specified by Move.
  • Door parts behave the same as parts in Geometry (collisions, projectiles, etc.).
  • Each door model must contain:
    • PrimaryPart (under Pivot in model properties)—this is the part that moves when the door opens or closes. It must be Anchored, but need not be visible. See photos on the left for details.
    • Trigger—detects player input to open the door. A model can have multiple Triggers with different settings, or none (auto–opens/closes by stage). Triggers use a Configuration named “Properties” with these attributes:
      • ActivationType (1–4):
        • 1 = touch (walking into Trigger). Disables CanCollide, acts like Ignore.
        • 2 = melee. Enables CanCollide, acts like Geometry.
        • 3 = melee, bullets, projectiles. Enables CanCollide, acts like Geometry.
        • 4 = other behavior (if present).
      • Duration: how long the door stays open (in seconds; excludes MoveTime).
      • PressMax: how many times the Trigger can be used per round (–1 = unlimited).
      • Team: “GRN”, “RED”, or blank for both teams.
      • WaitForFinish: if enabled, forces players to wait until the door fully closes before retriggering.
    • ForceCloseDuring Configuration:
      • Specifies which stages the door cannot be opened by players.
      • “Setup” = only during AD setup phase. “Stage1” = before any captures. “Stage2/3” = after captures in AD/KOTH/3CP as appropriate.
    • ForceOpenDuring Configuration:
      • Specifies which stages the door stays permanently open.
      • Follows the same stage naming as ForceCloseDuring.
      • Cannot overlap with the same stage in ForceCloseDuring.

Clips

  • The Clips folder contains invisible walls blocking players.
  • Clips parts are ignored by weapons but block players.
  • CanCollide must be enabled.

Items

  • The Items folder has resupply cabinets and health/ammo kits.
  • Models match in-game sizes—use them!
  • Parts named “Resupply” spawn cabinets; kits named SizeType (“MediumAmmo”, “SmallHP”).

ControlPoints

  • Holds control objects for KOTH, AD, 3CP, Arena.
  • KOTH/Arena use “ControlPoint” for the hill.
  • AD: ControlPoint1 then ControlPoint2.
  • 3CP: CP1=Red start, CP2=neutral, CP3=Green start.
  • Each has PlayerCaptureTime, PointLocation, CaptureZone.

Briefcases

  • For CTF only: “Green” & “Red” objectives.
  • Objectives are any part named accordingly.
  • GreenTrigger / RedTrigger fire on touch by opposing carrier.

TeamSpecific

  • Per-team folders: GRN & RED with Spawns, Checkpoints, SpawnZones, Barriers.
  • Spawns: respawn points per stage.
  • Checkpoints: optional respawns; kill opposing team on touch.
  • SpawnZones: class/loadout zones; opposing zone = kill.
  • Barriers: block only opposite team.

Settings

  • Contains Lighting, Terrain, and General settings.
  • Use the TC2 Map Plugin to automate Lighting & Terrain.
  • General toggles (AllowUnanchoredParts, Climbing) off unless needed.

GameMode

  • GameMode sets gamemode: "KOTH","AD","3CP","CTF","Arena","None".
  • KOTH: adds ActivateIn IntValue (seconds until hill capture).
  • AD: adds GameTimer, FirstCap, SetupTimer.
  • 3CP: no extra values.
  • CTF: adds CritsOnCap, MaxTime, PlayTo, TimeOnCap.
  • Arena: adds ActivateIn, PrepTime.
  • None: only PrepTime (infinite timer; end via VIP command).
